
### 1. 用户注册接口
- **URL**: `/register`
- **请求方法**: `GET`, `POST`
- **功能描述**: 用户注册功能，支持 JSON 和表单格式的请求。
- **请求参数**:
  - **JSON 请求**:
    - `username` (str, required): 用户名
    - `password` (str, required): 用户密码
  - **表单请求**:
    - `username` (str, required): 用户名
    - `password` (str, required): 用户密码
    - `confirm_password` (str, required): 确认密码
- **响应数据**:
  - **JSON 请求**:
    - 成功注册: `{'message': '1'}`，状态码 201
    - 输入信息不完整: `{'message': '2'}`，状态码 400
    - 用户已存在: `{'message': '3'}`，状态码 400
    - 注册失败: `{'message': '错误信息'}`，状态码 500
  - **表单请求**:
    - 两次输入密码不一致: 重定向到注册页面，提示 "两次输入的密码不一致"
    - 用户已存在: 重定向到注册页面，提示 "用户名已存在"
    - 注册成功: 重定向到登录页面
    - 注册失败: 重定向到注册页面，提示 "注册失败，请稍后重试"

### 2. 用户登录接口
- **URL**: `/login`
- **请求方法**: `GET`, `POST`
- **功能描述**: 用户登录功能，支持 JSON 和表单格式的请求。
- **请求参数**:
  - **JSON 请求**:
    - `username` (str, required): 用户名
    - `password` (str, required): 用户密码
  - **表单请求**:
    - `username` (str, required): 用户名
    - `password` (str, required): 用户密码
    - `remember` (str, optional): 是否记住登录状态
- **响应数据**:
  - **JSON 请求**:
    - 登录成功: `{'message': '1', 'user': {'id': 用户 ID, 'username': 用户名, 'is_admin': 是否为管理员}}`，状态码 200
    - 输入信息不完整: `{'message': '2'}`，状态码 400
    - 用户名或密码错误: `{'message': '0'}`，状态码 401
    - 登录失败: `{'message': '错误信息'}`，状态码 500
  - **表单请求**:
    - 登录成功:
      - AJAX 请求: `{'success': true, 'redirect': 重定向 URL, 'is_admin': 是否为管理员}`
      - 普通请求: 重定向到首页
    - 用户名或密码错误:
      - AJAX 请求: `{'success': false, 'message': '用户名或密码错误'}`，状态码 401
      - 普通请求: 重定向到登录页面，提示 "用户名或密码错误"
    - 登录失败: 重定向到登录页面，提示 "登录过程中出现错误，请稍后重试"

### 3. 用户注销接口
- **URL**: `/logout`
- **请求方法**: `GET`
- **功能描述**: 用户注销登录，需要用户已登录。
- **请求参数**: 无
- **响应数据**: 重定向到首页

### 4. 药材检测接口
- **URL**: `/detect`
- **请求方法**: `GET`, `POST`
- **功能描述**: 上传图片进行药材检测，返回检测结果和相似药材推荐。
- **请求参数**:
  - **POST 请求**:
    - `image` (file, required): 上传的图片文件
- **响应数据**:
  - 上传文件为空: 重定向到检测页面，提示 "没有选择文件"
  - 文件类型不支持: 重定向到检测页面，提示 "不支持的文件类型"
  - 检测成功: 渲染 `result.html` 页面，包含药材信息、置信度、图片路径、相似药材数据和百度 AI 识别结果
  - 检测失败: 重定向到检测页面，提示 "识别过程中出现错误: 错误信息"

### 5. 获取药材详情接口
- **URL**: `/api/herb/<int:herb_id>`
- **请求方法**: `GET`
- **功能描述**: 根据药材 ID 获取药材的详细信息，需要用户已登录。
- **请求参数**:
  - `herb_id` (int, required): 药材的 ID
- **响应数据**:
  - 成功获取: `{
    'id': 药材 ID,
    'name': 药材名称,
    'english_name': 英文名称,
    'photo_number': 照片数量,
    'page_number': 页码,
    'description': 描述信息,
    'effects': 功效,
    'properties': 性质,
    'similar_herbs': 相似药材,
    'usage_parts': 使用部位,
    'image_path': 图片路径,
    'season': 季节
  }`
  - 获取失败: `{'error': '获取药材详情失败'}`，状态码 500

### 6. 获取检测历史详情接口
- **URL**: `/api/history/<int:record_id>`
- **请求方法**: `GET`
- **功能描述**: 根据检测记录 ID 获取检测历史的详细信息，需要用户已登录，且只能访问自己的历史记录。
- **请求参数**:
  - `record_id` (int, required): 检测记录的 ID
- **响应数据**:
  - 无权访问: `{'error': '无权访问此记录'}`，状态码 403
  - 成功获取: `{
    'id': 记录 ID,
    'image_path': 图片路径,
    'detection_time': 检测时间,
    'confidence': 置信度,
    'herb': {
      'id': 药材 ID,
      'name': 药材名称,
      'properties': 性质,
      'effects': 功效,
      'usage_parts': 使用部位,
      'description': 描述信息,
      'image_path': 图片路径
    },
    'similar_herbs': [
      {
        'id': 相似药材 ID,
        'name': 相似药材名称,
        'properties': 性质,
        'effects': 功效,
        'usage_parts': 使用部位,
        'image_path': 图片路径
      }
    ]
  }`

### 7. 获取传感器实时数据接口
- **URL**: `/api/sensor/realtime`
- **请求方法**: `GET`, `POST`
- **功能描述**: 获取传感器的实时数据，支持 POST 请求指定查询的传感器和数量。
- **请求参数**:
  - **POST 请求**:
    - `sensors` (list, optional): 要查询的传感器列表
    - `sensor_num` (int, optional): 查询数量，默认 10 条
- **响应数据**:
  - **POST 请求**:
    - 暂无传感器数据: `{'success': false, 'message': '暂无传感器数据'}`，状态码 404
    - 处理请求失败: `{'success': false, 'message': '处理请求失败: 错误信息'}`，状态码 500
    - 成功获取:
      - 指定传感器: `{传感器名称: 传感器值, 'event_time': 事件时间}`
      - 全部传感器: `{
        'temp': 温度,
        'humi': 湿度,
        'sound': 声音,
        'tilt': 倾斜度,
        'vibrate': 振动,
        'fire': 火灾,
        'smoke': 烟雾,
        'light_sense': 光照,
        'event_time': 事件时间
      }`
  - **GET 请求**:
    - 暂无传感器数据: `{'success': false, 'message': '暂无传感器数据'}`，状态码 404
    - 获取失败: `{'success': false, 'message': '获取传感器数据失败'}`，状态码 500
    - 成功获取: `{
      'success': true,
      'data': {
        'temperature': 温度,
        'humidity': 湿度,
        'sound': 声音,
        'tilt': 倾斜度,
        'vibrate': 振动,
        'fire': 火灾,
        'smoke': 烟雾,
        'light_sense': 光照,
        'event_time': 事件时间
      }
    }`

### 8. 获取传感器历史数据接口
- **URL**: `/api/sensor/history`
- **请求方法**: `GET`
- **功能描述**: 根据时间范围和数量限制获取传感器的历史数据。
- **请求参数**:
  - `start_time` (str, optional): 开始时间，格式 `YYYY-MM-DD HH:MM:SS`
  - `end_time` (str, optional): 结束时间，格式 `YYYY-MM-DD HH:MM:SS`
  - `limit` (int, optional): 查询数量，默认 100 条
- **响应数据**:
  - 获取失败: `{'success': false, 'message': '获取历史数据失败'}`，状态码 500
  - 成功获取: `{
    'success': true,
    'data': [
      {
        'temperature': 温度,
        'humidity': 湿度,
        'sound': 声音,
        'tilt': 倾斜度,
        'vibrate': 振动,
        'fire': 火灾,
        'smoke': 烟雾,
        'light_sense': 光照,
        'event_time': 事件时间
      }
    ],
    'total': 数据总数
  }`

### 9. 获取传感器统计数据接口
- **URL**: `/api/sensor/statistics`
- **请求方法**: `GET`
- **功能描述**: 根据指定的天数获取传感器的统计数据。
- **请求参数**:
  - `days` (int, optional): 查询天数，默认 7 天
- **响应数据**:
  - 暂无记录: `{
    'success': true,
    'data': {
      'temperature': {'avg': 0, 'max': 0, 'min': 0},
      'humidity': {'avg': 0, 'max': 0, 'min': 0},
      'light': {'avg': 0, 'max': 0, 'min': 0}
    }
  }`
  - 成功获取: `{
    'success': true,
    'data': {
      'temperature': {'avg': 平均温度, 'max': 最高温度, 'min': 最低温度},
      'humidity': {'avg': 平均湿度, 'max': 最高湿度, 'min': 最低湿度},
      'light': {'avg': 平均光照, 'max': 最高光照, 'min': 最低光照}
    }
  }`
