# 第三章 系统设计

## 3.1 系统总体架构

本系统采用基于深度学习和人工智能技术的中药材识别方案，通过整合本地深度学习模型和百度AI云端识别能力，构建了一个高效、准确的中药材识别系统。系统采用Flask作为后端框架，TensorFlow作为深度学习引擎，通过双重识别引擎相互验证，显著提高了中药材识别的准确性和可靠性。

### 3.1.1 系统设计目标

本系统的设计目标主要包括以下几个方面：

1. 提供准确的中药材识别服务，满足中医药相关人员和爱好者的实际需求；
2. 通过双重识别引擎提高识别准确率，降低误识别率；
3. 构建用户友好的交互界面，提供便捷的操作体验；
4. 实现系统的可扩展性和可维护性，支持后续功能扩展。

### 3.1.2 系统架构设计

系统采用经典的三层架构设计，包括表示层、业务逻辑层和数据访问层，具体架构如图3-1所示。

## 3.2 系统功能模块设计

### 3.2.1 用户管理模块

用户管理模块主要实现用户信息的注册、登录、修改等功能，具体包括：

1. **用户注册功能**
   - 基本信息录入：用户名、密码、邮箱等
   - 信息验证：格式验证、重复性检查
   - 账号激活：邮件验证机制

2. **用户登录功能**
   - 身份认证：用户名密码验证
   - 会话管理：登录状态维护
   - 安全控制：密码加密存储

3. **个人信息管理**
   - 信息修改：密码更新、个人资料维护
   - 偏好设置：系统使用偏好配置
   - 安全退出：会话安全终止

### 3.2.2 药材识别模块

药材识别模块是系统的核心功能模块，采用双重识别引擎设计，具体包括：

1. **图像采集与预处理**
   - 图像获取：支持拍摄、上传、视频提取等多种方式
   - 图像标准化：尺寸调整、颜色校正
   - 图像增强：对比度调整、噪声消除

2. **双重识别引擎**
   - 本地模型识别：基于EfficientNetV2B0架构的深度学习模型
   - 云端识别：调用百度AI通用物体识别API
   - 结果融合：基于置信度的结果整合算法

3. **识别结果处理**
   - 结果展示：识别结果可视化展示
   - 相似药材推荐：基于多维特征的相似度计算
   - 详细信息展示：药材属性、功效等信息的展示

### 3.2.3 数据管理模块

数据管理模块负责系统数据的存储、查询和管理，具体包括：

1. **识别历史管理**
   - 历史记录存储：识别结果的持久化存储
   - 记录查询：多维度检索功能
   - 数据导出：支持多种格式的数据导出

2. **药材信息管理**
   - 药材库维护：药材信息的增删改查
   - 数据关联：药材间关系的维护
   - 数据导入导出：批量数据处理

3. **系统数据管理**
   - 数据备份：定期自动备份机制
   - 数据恢复：备份数据的恢复功能
   - 系统日志：操作日志记录与分析

## 3.3 数据库设计

### 3.3.1 数据库选型

本系统采用SQLite作为数据库管理系统，主要考虑以下因素：

1. 轻量级：适合中小规模应用
2. 零配置：无需复杂的配置过程
3. 跨平台：支持多种操作系统
4. 性能稳定：满足系统性能需求

### 3.3.2 数据表设计

系统主要包含以下数据表：

1. **用户表(users)**
   ```sql
   CREATE TABLE users (
       id INTEGER PRIMARY KEY AUTOINCREMENT,
       username VARCHAR(50) NOT NULL UNIQUE,
       password VARCHAR(255) NOT NULL,
       email VARCHAR(100) NOT NULL UNIQUE,
       created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
       last_login TIMESTAMP,
       status INTEGER DEFAULT 1
   );
   ```

2. **药材表(herbs)**
   ```sql
   CREATE TABLE herbs (
       id INTEGER PRIMARY KEY AUTOINCREMENT,
       name VARCHAR(100) NOT NULL,
       alias TEXT,
       properties TEXT,
       effects TEXT,
       usage TEXT,
       image_path VARCHAR(255),
       created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
       updated_at TIMESTAMP
   );
   ```

3. **识别记录表(detection_records)**
   ```sql
   CREATE TABLE detection_records (
       id INTEGER PRIMARY KEY AUTOINCREMENT,
       user_id INTEGER,
       herb_id INTEGER,
       image_path VARCHAR(255),
       confidence FLOAT,
       detection_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
       FOREIGN KEY (user_id) REFERENCES users(id),
       FOREIGN KEY (herb_id) REFERENCES herbs(id)
   );
   ```

   - 敏感操作日志记录

### 3.4.2 数据安全

2. **数据备份**
